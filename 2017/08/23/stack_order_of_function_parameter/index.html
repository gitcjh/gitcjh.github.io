函数调用参数入栈顺序

发表于 2017-08-23 | 分类于 函数参数 | | 阅读次数
| 字数统计 604 | 阅读时长 3

之前一直以为函数的参数入栈顺序是从右向左，但事实上不同架构的系统实现是不同的。具体分析如下。

32位系统

32位系统：参数从右到左入栈

具体分析可看汇编程序，先给出测试代码

gdb列出汇编代码，如下

可以看到11-14行，汇编代码是先push $0x2然后push $0x1，最后才push %eax，这个是this指针。也就是说参数入栈顺序为后面的参数先入栈。

64位系统

64位系统稍微有点不同，前六个参数先放在寄存器中，其他参数从后面开始入栈。

测试代码如下

gdb汇编代码如下

  • 7-14行，后面的参数7和6先入栈，然后前面1-5以及this指针按照从右向左的顺序依次保存在寄存器中。
  • 23-29行，先将栈中的参数7和6压入，然后在按照1-6寄存器的顺序将寄存器中的参数压入。
